Programming Languages
TERM DEFINITION
Machine Language Sequences of instructions that are encoded as numeric digits to write a program.
Debugging Locating and correcting errors in the machine language.
Assembler Programs developed to convert mnemonic expressions into machine language instructions.
Assembly Language A mnemonic system for representing programs.
Translator A program that translates programs expressed in high level primitives into machine language programs.
Compiler Translator program having to compile several machine instructions into short sequences to simulate the activity requested by a single high level primitive.
Interpreter Programs that executed the instructions as they were translated instead of recording the translated version for future use.
Cross-platform Software Software that is independent of an operating system's design as well as the machine's hardware design and it therefore executable throughout a network.
Natural Language Languages that evolved over time without formal grammatical analysis such as English, German, and Latin.
Formal Language Languages precisely defined by grammars such as programming language.
Machine Independence Programming languages that did not rely on the characteristics of a particular machine.
High-level programming language Advancement in programming languages enabling construction of increasingly complex software system functions.
Object-oriented Programming (OOP) Software system viewed as a collection of units, ech capable of performing the actions related to itself as well as requesting actions of other objects. The objects interact to solve the problem.
Scripting Language Languages typically used to perform administrative tasks rather than develop complex programs.
Data Types:
  • Integer - Numeric data consisting of whole numbers, probably stored using two's complement notation.
  • Float - Numeric data that might contain values other than whole numbers, probably stored in floating-point notation.
  • String - A pattern of data
Primitive Data Type Data types that are included as primitives in programming language, such as int for integer and char for character.
Python Collection Data Structures:
  • List - Used to store multiple items in a single variable. Created using square brackets
  • Tuple - Used to store multiple items in a single variable. Collection which is ordered and unchangeable. Created using round brackets.
  • Set - Used to store multiple items in a single variable. Collection which is unordered, unchangeable, and unindexed.
  • Dictionary - Used to store data values in key: value pairs. Collection which is ordered, changeable, and do not allow duplicates.
Function A set of instructions for performing a task that can be used as an abstract tool by other program units.
Local Variable A variable declared within a function, meaning it can only be referenced within that function.
Global Variable Variables whose scopes are not restricted to a particular part of a program.
Source Program Program in its original form.
Object Program Translated version of the program.

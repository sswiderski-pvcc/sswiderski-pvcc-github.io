Software Engineering
TERM DEFINITION
software engineering The engineering process to find principles that guide the software development process and lead to efficient, reliable software products.
integrated development environment (IDE) Systems that combine tools for developing software into a single, integrated package.
Association for Computing Machinery (ACM)
    Purpose of ACM: scientific and educational organization dedicated to advancing the arts, sciences, and applications of information technology.
    Benefits of ACM membership: providing opportunities for life-long learning, career development, and professional networking
    Student ACM membership: students enhance their learning through the exchange of ideas with other students and established professionals
Institute of Electrical and Electronics Engineers (IEEE)
    Purpose of IEEE: To foster technological innovation and excellence for the benefit of humanity
    Benefits of IEEE membership: help shape your career, offer resources to acquire new skills, and advance your professional development.
    Student IEEE membership: Career resources and recognition, professional networking,continuing education, and discounts.
Software Life Cycle: Development Phase
    Requirements analysis: To specify what services the proposed system will provide, to identify any conditions on those services, and to define how the outside world will interact with the system.
    Design: Involves creating a plan for the construction of the proposed system.
    Implementation: Involves the actual writing of programs, creation of data files, and development of databases.
    Testing: Process of debugging programs and confirming that the final software product was compatitable with the software requirements specification.
modular implementation Separating a program's functions into independent pieces or building blocks, each containing all the parts needed to execute a single aspect of the functionality.
Unified Modeling Language (ULM) A general purpose modeling language that is intended to provide a standard way to visualize the design of a system.
use case interactions between the system and its users
use case diagram Attempts to capture the image of the proposed system from the user's point of view.
Data relationships
    one-to-one: each primary key value relates to only one record.
    one-to-many: The primary key table contains only one record that relates to one or many records.
    many-to-many: Each record in a table can relate to any number of records in another table.
design pattern A predeveloped model for solving a recurring problem in software design.
Software quality assurance (SQA) Groups charged with overseeing and enforcing the quality control systems adopted by the organization.
Software testing
    glass box testing: Software tester is aware of the interior structure of the software and uses this knowledge when designing the test.
    black box testing: Test that do not rely on knowledge of the software's interior composition.
    beta testing: When a preliminary version of the software is given to a segment of the intended audience with the goal of learning how the software performs in real-life situations before the final version of the product is solidified and released to the market.
    alpha testing: Similar testing to beta but performed at the developer's site.
Documentation
    user documentation: To explain the features of the software and describe how to use them.
    system documentation: To describe the software's internal composition so that the software can be maintained later in its life cycle.
    technical documentation: To describe how a software system should be installed and serviced.
Intellectual property law
    software license: A legal agreement between the owner and user of a software product that grants the user certain permissions to use the product without transferring ownership rights to the intellectual property.
    software disclaimer: Software developers use disclaimer to state the limitations of their liability.
    best defenses against software liability claims is to apply sound software engineering principles during the software's development, to use a level of care compatible with the software's application, and to produce and maintain records that validate these endeavors.

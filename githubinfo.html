All About GitHub

Q. What is a version control system?
A. A version control system (VCS) tracks changes to a file or set of files over time.
Q. What is Git?
A. Git is a distributed, open-source version control system (VCS) that enables you to store code, track revision history, merge code changes, and revert to earlier code version when needed.
Q. What is GitHub?
A. GitHub, Inc. is an Internet hosting service for software development and version control using Git. It provides the distributed version control of Git plus access control, bug tracking, software feature requests, task management, continuous integration, and wikis for every project.
Q. What is the difference between Git and GitHub?
A. Git is a version control system that lets you manage and keep track of your source code history. GitHub is a cloud-based hosting service that lets you manage Git repositories.
Q. Who started GitHub and how was it started?
A. The site was launched in April 2008 by Tom Preston-Werner, Chris Wanstrath, and PJ Hyett. GitHub was created to provide developers the option to host code securely and manage commits to code in proper manner.
Q. What company owns it now?
A. Microsoft
Q. How much does a GitHub account cost?
A. GitHub is free for personal accounts
Q. What is the Octocat?
A. Octocat is the mascot for GitHub. It is part cat part octopus

Git/GitHub terms:

repository
- a central location in which data is stored and managed.
commit
- a snapshot of your repository
fork
- a new repository that shares code and visibility settings with the original “upstream” repository
push
- uploads all local branch commits to the corresponding remote branch
pull requests
- let you tell others about changes you've pushed to a GitHub repository
workflows
- a recipe or recommendation for how to use Git to accomplish work in a consistent and productive manner
issues
- allows you to create text-based descriptions of tasks, bugs, changes, and updates in your software development project
raw button
- Allows you to view or copy the raw content of a file without any styling
blame button
- used to examine the contents of a file line by line and see when each line was last modified and who the author of the modifications was
